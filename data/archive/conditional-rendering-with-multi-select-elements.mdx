In Salesforce OmniStudio, developers often need to conditionally render elements based on user selections in multi-select fields within OmniScripts. This functionality enhances user experience by displaying relevant information dynamically. Here's a guide on how to implement conditional rendering using multi-select elements.

**1. Understanding Multi-Select Elements in OmniScripts**

Multi-select elements allow users to choose multiple options from a list. In OmniScripts, you can add these elements to enable such functionality. To incorporate a multi-select element:

- Navigate to your OmniScript in the OmniStudio app.
- Drag and drop the **Multi-Select** element from the component palette into your script.
- Configure the options and settings as needed.

![Button Properties](/static/images/multi-select.png)


**2. Setting Up Conditional Rendering**

To conditionally display elements based on selections in a multi-select field, follow these steps:

- **Identify the Element to Control**: Determine which element(s) should appear or hide based on the user's selections.
- **Configure the Conditional View Property**: Use the **Conditional View** property of the target element to set the criteria for its visibility.

For example, if you want an element to display when a specific option is selected in the multi-select field, set the **Conditional View** property accordingly. Salesforce's guide on [Using the Conditional View Property](https://help.salesforce.com/s/articleView?id=sf.os_conditionally_display_elements_using_the_conditional_view_property_19389.htm&language=en_US&type=5) provides detailed instructions.

**3. Considerations and Best Practices**

When implementing conditional rendering with multi-select elements, keep the following in mind:

- **Use of Operators**: The **CONTAINS** operator is essential for checking if a specific value exists within a multi-select field. However, some users have noted the absence of this operator in certain contexts, which can limit functionality. For more details, see this [Salesforce Idea](https://ideas.salesforce.com/s/idea/a0B8W00000PiyiZUAR/add-contains-operator-to-omniscript-conditional-view).
- **Performance Considerations**: Ensure that complex conditional logic does not adversely affect the performance of your OmniScript. Test thoroughly to maintain a smooth user experience.
- **User Experience**: Clearly indicate to users which selections will trigger additional fields or actions to avoid confusion.
